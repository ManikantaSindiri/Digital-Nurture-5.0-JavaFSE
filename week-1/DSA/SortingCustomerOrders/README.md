# Exercise 3: Sorting Customer Orders

## Scenario
You are tasked with sorting customer orders by their total price on an e-commerce platform. This helps in prioritizing high-value orders.

## 1. Understand Sorting Algorithms
- Learn about common sorting algorithms such as Bubble Sort, Selection Sort, and Merge Sort.
- Understand how comparison-based sorting works.
- Think about why sorting orders is useful for processing and prioritization.

## 2. Setup
- Create a new folder for this exercise inside the DSA section.
- Prepare a Java program that stores customer orders with a total price.
- Decide how the orders should be sorted.

## 3. Implementation
- Implement a sorting feature that can:
  - store customer order details
  - sort orders by total price
  - display the sorted orders

## 4. Analysis
- Compare the time complexity of different sorting methods.
- Explain when a simple sorting algorithm is enough and when a more efficient one is better.
- Discuss how sorting improves workflow in real e-commerce systems.

## Expected Learning Outcome
By completing this exercise, you will understand how sorting algorithms help organize and prioritize data efficiently.
