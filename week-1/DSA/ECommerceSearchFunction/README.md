# Exercise 2: E-commerce Platform Search Function

## Scenario
You are working on the search functionality of an e-commerce platform. The search needs to be optimized for fast performance.

## 1. Understand Asymptotic Notation
- Learn the meaning of Big-O, Big-Omega, and Big-Theta.
- Understand how time complexity helps compare search approaches.
- Think about why fast search matters in large product databases.

## 2. Setup
- Create a new folder for this exercise inside the DSA section.
- Prepare a Java program to simulate a product search feature.
- Choose a data structure suitable for quick lookup, such as a HashMap or ArrayList.

## 3. Implementation
- Implement a search feature that can:
  - store product names or IDs
  - search for a product quickly
  - display the result if found
- Example operations:
  - add products
  - search by product name
  - search by product ID

## 4. Analysis
- Compare the search performance of different approaches.
- Explain why a HashMap provides faster lookup than scanning an ArrayList linearly.
- Discuss the trade-offs in memory and implementation complexity.

## Expected Learning Outcome
By completing this exercise, you will understand how asymptotic notation and data structures improve the efficiency of search operations in real-world systems.
